점수 시스템

매매 결정 로직

5개 지표 합산 → 임계값 비교 → ignore / notify / auto_trade

지표조건점수
뉴스 감성매우 긍정 (FinBERT)+7 max
RSI과매도 < 30+3
RSI과매수 > 70-3
MACD골든크로스+5
MACD데스크로스-5
거래량> 150% 평균+4
거래량50~150% 평균+2
거래량< -50% 평균-2
공포/탐욕극도의 공포 < 25+2
공포/탐욕극도의 탐욕 > 75-2

임계값 및 행동

절댓값 기준
0 ~ 9 무시 (IGNORE)
10 ~ 19 사용자 승인 요청 (NOTIFY)
20 이상 자동 거래 실행 (AUTO_TRADE)
양수 = 매수 신호, 음수 = 매도 신호
SCORE_NOTIFY_MIN, SCORE_AUTO_TRADE_MIN으로 .env에서 조정 가능

🔥 Phase 2 — 데이터 수집 자동화 최우선

📰
RSS 뉴스 자동 수집기 High
CoinDesk · CoinTelegraph · Binance News RSS 파싱. feedparser 이미 requirements에 포함. services/news_collector.py 구현 필요. 중복 URL 필터링 포함.
⏰
분석 스케줄러 High
APScheduler 또는 asyncio 루프로 X분마다 RSS 수집 → /analyze 자동 실행. 현재는 OpenClaw에서 수동 POST 필요.
😱
Fear & Greed Index 실시간 연동 High
alternative.me API로 실시간 값 조회. 현재 /analyze 요청 시 수동 입력. 자동 조회 서비스 구현 필요.
🦀
OpenClaw 웹훅 수신 엔드포인트 Medium
OpenClaw(현재 18789 포트 실행 중)에서 뉴스 수집 후 POST로 전달받는 /webhook/news 엔드포인트 구현.

📬
알림 시스템 Medium
score ≥ 10 (NOTIFY)일 때 사람에게 알림. OpenClaw 메시지 우선. Telegram Bot API 또는 Discord Webhook 추가 후보. services/notifier.py 구현.
🔬
백테스팅 시스템 Medium
backtest/ 디렉토리 구현. Bybit 히스토리컬 데이터 다운로드 → 과거 뉴스 데이터로 전략 시뮬레이션. Backtrader 또는 자체 구현. 수익률 · MDD · 샤프 비율 계산.
🧪
Testnet 실거래 검증 Medium
현재 Testnet API 키 설정 완료. MOCK_MODE=false로 전환 후 실제 Testnet 주문 테스트. 주문 체결 · 잔고 조회 · 에러 처리 검증.
📉
일일 손실 추적기 Low
현재 MAX_DAILY_LOSS_PCT=3% 설정은 있으나 실제 추적 로직 미구현. trade_logs를 집계해 일일 손실이 한도 초과 시 자동 거래 중지 기능 필요.

🤖
FinBERT → DistilRoBERTa 전환 테스트 Low
