⚡ Phase 4 — 구현 상세

1. 동적 포지션 사이징 (Dynamic Position Sizing)

기존: qty = 0.001 하드코딩 → 신호 강도·자산 규모에 맞는 수량 자동 계산

항목공식 / 설명예시 (BTCUSDT, equity 1,000 USDT)
기본 투자금equity × MAX_POSITION_PCT%1,000 × 10% = 100 USDT
점수 배수min(|score| / 20, 1.5)score=20 → 1.0×, score=30 → 1.5×
실제 투자금기본 투자금 × 점수 배수score=20 → 100 USDT, score=30 → 150 USDT
주문 수량투자금 / 현재 가격100 / 63,500 ≈ 0.001575 BTC

2. 연속손실 실시간 추적

이벤트동작거래 차단 조건
주문 성공consecutive_losses 카운터 초기화-
주문 실패consecutive_losses += 1≥ MAX_CONSECUTIVE_LOSSES (기본 5회)
손실 PnLdaily_loss_usdt += |pnl|일일 손실 ≥ MAX_DAILY_LOSS_PCT % (기본 3%)
매일 자정daily_loss_usdt 자동 초기화-
수동 리셋POST /api/v1/risk/resettrading_halted → false

⚠️ 주의사항

  • 🔴 백테스트 성능 ≠ 실제 수익 — 과적합(Overfitting) 위험 항상 존재
  • 🔴 AI 모델 환각(Hallucination) — FinBERT 판단 오류 가능성 존재
  • 🟡 실거래 전 Testnet 필수 검증 — BYBIT_TESTNET=true 상태에서 충분히 테스트
  • 🟡 반자동 모드 권장 — SCORE_AUTO_TRADE_MIN을 높게 설정해 사람이 최종 승인
  • 🟢 Mock 모드로 먼저 — MOCK_MODE=true 상태에서 전체 파이프라인 충분히 익힌 후 전환
